window对象

  • 定义
  1. 全局对象。
  2. 访问浏览器窗口的API
  • 标准问题

早期没规范，标准不统一。后期BOM主要部分纳入H5标准。

  • 全局作用域

所有在全局作用域中定义的属性和方法都成为window的属性和方法

框架对象

在有框架的页面中，window将框架保存在frames集合中。可以通过下标或者框架的name属性作为key值来进行访问操作。

  • 框架之间的关系

这里只要是指父框架和子框架之间的关系。

  1. 在父级页面中——window对象可以通过frames集合来访问子框架。
  2. 子框架页面中——有top、parent两个对象来访问父级页面
  • 访问特定框架

窗口与页面文档

  • 窗口位置
    关于浏览器的位置与大小各个浏览器厂商在实现上没有统一。在取值标准和提供的API接口上都有不同。

关于位置

  1. IE和opera定义的是屏幕到window对象可见区域的距离，返回的值精确

  2. chrome 、safari 、Firefox则是屏幕到浏览器窗口的距离，而且在各种情况下返回的值并不精确

  3. IE、safari、 opera chrome 提供的是screenLeft screenTop

  4. Firefox则是screenX screenY

窗口大小
关于窗口大小的属性有四个：innerWidth innerHight outerWidth outerHight。

  1. IE8及以下版本不提供
  2. chrome中四个属性返回的都是视口大小
  3. 在safari 、Firefox IE9+中outerWidth outerHight 返回浏览器窗口大小
  4. opera的标准又是其它

本人自己测试过（2017），在新版的 Firefox IE10 chrome以及opera中现在这四个值都已经统一，
outer组：表示浏览器窗口大小（其中ie和火狐浏览器在窗口最大化的时候边框被隐藏了，但是计算出来的大小还是包括边框的，所以最大化的值是超过屏幕大小的）
inner组：表示视口大小

总之最后无法在跨浏览器的情况下获取精确的窗口位置及大小值。

系统对话框 alert、confirm及prompt

alert
警告框，直接输入要显示的值
confirm
确认框，返回true（确认）或false（取消）
prompt
提示框，提示用户输入相应的文本，返回用户输入的文本

setTimeout和 setInterval 方法

setTimeout 超时调用，只执行一次。clearTimeout来取消
setInterval 定时执行 clearInterval来取消

localtion对象

这个对象提供了与当前加载文档相关的信息。包括主机名、端口号 url 路径名 协议等等

位置操作

  • assign方法

    • 打开一个新的url页面——产生历史记录
    • 一般有window.localtion和localtion.href调用

  • replace方法

导航到一个新的url——覆盖当前页面的历史记录

  • reload
  1. reload()刷新页面 —— 有缓存则用缓存
  2. reload(true) ——强制向向服务器请求刷新页面
