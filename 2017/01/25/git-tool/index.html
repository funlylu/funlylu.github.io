git merge

git merge的基本用法为把一个分支或或某个commit的修改合并到现在的分支上。

在提交代码的时候经常会遇到这样的情况：
你commit了你的修改，正准备提交的时候—-显示失败了。。。。你的本地版本不是最新的
远程服务器上面有好几个新的commit。

解决办法：

  1. 你们分工明确，各自负责的代码没有冲突

    • pull服务器上面最新的代码与你本地的代码merge就好。
      一般的你pull的时候，会提示你输入merge的信息，直接输入即可。

  2. 各自修改的是同一个文件，之间有冲突

    • clone一分新的代码到本地
    • 文档比较工具，手动合并
    • 重新提交。
