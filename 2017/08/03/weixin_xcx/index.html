年前微信小程序一出来就特别的火爆。博主当时也进行了一番研究，当时没有深入的探索所以没有写文档。今天博主闲来无事，来细细研究一下这个微信小程序的开发流程。

申请微信小程序账号

  1. 打开微信公众号官网,注册
  2. 注册类型 选择“小程序”，然后按照步骤填写相关信息即可

用账号登录这个微信公众平台。这个平台就是用来管理我们开发的微信小程序的。里面有小程序开发的文档的链接。

获取微信小程序的 AppID

  1. 打开微信公众号官网 登录
  2. copy这个小程序ID，到时候创建项目的时候要用

开发环境搭建

我们需要微信开发者工具来完成微信小程序的开发。

  1. 开发者工具
  2. 安装完成之后用自己的微信扫描登录即可
  3. 选择新建项目

  1. 填写项目相关信息，把上面那个AppID填写上

  2. 点击新建后工具会为我们生成小程序的代码框架简易教程，里面包含了一些简单的页面文件。

主要文件分析

点击开发者工具左侧导航的“编辑”，我们可以看到这个项目，已经初始化并包含了一些简单的代码文件。最关键也是必不可少的，是 app.js、app.json、app.wxss 这三个。其中，.js后缀的是脚本文件，.json后缀的文件是配置文件，.wxss后缀的是样式表文件。微信小程序会读取这些文件，并生成小程序实例。

  1. app.js

是小程序的脚本代码。我们可以在这个文件中监听并处理小程序的生命周期函数、声明全局变量。调用框架提供的丰富的 API，如本例的同步存储及同步读取本地数据。想了解更多可用 API，可参考 API 文档

  1. app.json

是对整个小程序的全局配置。我们可以在这个文件中配置小程序是由哪些页面组成，配置小程序的窗口背景色，配置导航条样式，配置默认标题。注意该文件不可添加任何注释。更多可配置项可参考配置详解

  1. app.wxss

是整个小程序的公共样式表。我们可以在页面组件的 class 属性上直接使用 app.wxss 中声明的样式规则。

运行及真机预览

点击 【 项目->预览 】会生成一个二维码 用微信扫描即可在微信上预览

相关技术须知

  1. 开发微信小程序有自己的一套 【框架】
  2. 有一套微信风格的布局组件
  3. 框架提供丰富的微信原生API，可以方便的调起微信提供的能力，如获取用户信息，本地存储，支付功能等

使用限制

  1. 脚本中没有window对象
  2. zepto/jquery 无法使用
  3. 一个应用同时只能打开5个页面，当已经打开了5个页面之后，wx.navigateTo不能正常打开新页面
  4. 样式表不支持级联选择器，支持类，和标签选择器
  5. 本地资源无法通过 WXSS 获取

例如：background-image：可以使用网络图片，或者 base64，或者使用<image/>标签
