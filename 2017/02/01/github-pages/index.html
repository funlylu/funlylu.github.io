GitHub Pages功能使用总结

Feb 1, 2017 | 工具 | 阅读

Github是一个Git 代码仓库托管平台，本文不讲代码托管功能，我们来说说它GitHub Pages(网站服务功能)。

GitHub Pages简介

GitHub Pages它是基于在代码功能其实网站服务器一样。你可以通过浏览器来访问你托管在GitHub上面的web项目代码。利用这个功能我们可以创建免费的个人博客（比较简陋的静态页面，毕竟都是免费的），前端demo演示等等。
GitHub Pages分为——用户/企业网站和项目网站。项目网站创建的前提是必须创建了——用户/企业网站。
用户/企业网站的创建方法一样。下面我们用创建个人网站的创建来展示。

如何创建 ————个人博客网站

  1. 首先你要创建一个名称为username.github.io（username是你Github用户名）
  2. 上传你的个人博客项目静态代码（可以用hexo框架搭建，不知道hexo？自行百度）
  3. 在该项目的Settings部分找到GitHub Pages
    • Source: 选择一个分支作为博客网站展示的内容
    • Theme Chooser: 网站主题样式——github会为 你的网站加上主题样式。（可忽略）
    • Custom domain: 为博客设置域名-如果不想用默认的域名的话。(可忽略)
    • Enforce HTTPS: 默认选上-你也可以去掉(可忽略)

访问URL：此时就可以通过username.github.io这个默认域名来访问你的博客啦。

如何创建项目网站

创建项目网站就简单了。

  1. 直接在项目的Settings部分找到GitHub Pages部分内容进行设置即可。一般选择要作为网页展示的分支就行。

访问URL：username.github.io/project 就可以访问项目页面了
