更新

  1. git pull 从远程获取最新版本并merge到本地

  2. git push 提交本地commit到远程仓库

  3. git fetch 相当于是从远程获取最新版本到本地，不会自动merge

撤销提交及恢复

撤销
  • 混合撤销

撤销commit和添加到暂存区的操作，源码还在。用git status 仍可以看到工作区已经修改但是未被缓存的文件

  • 软撤销

只撤销commit，暂存区不动

  • 硬撤销

把代码和commit以及暂存区都恢复到pre_commit_id 那次提交

恢复

git reset其实可以撤销也可以恢复，只要我们知道指定恢复的commit哈希值即可
